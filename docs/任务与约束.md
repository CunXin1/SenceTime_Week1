## 三、Mac 上的关键取舍决策(FAQ)

### Q1. Miniconda 能用 Docker 代替吗?
**不建议(在 Mac 上)。** Docker Desktop 在 Mac 上跑在 Linux 虚拟机里,**拿不到 GPU**——既无 CUDA,也**访问不到 MPS**,容器内一切推理都是纯 CPU,极慢。
- conda 原生跑在 macOS,PyTorch 可直接用 MPS 加速 → **Mac 现在用 conda**。
- Docker 的价值在**下周 4090(Linux)**:有 NVIDIA Container Toolkit 可 GPU 透传,适合环境复现。
- 结论:**不是二选一——Mac 用 conda,4090 上可选 Docker。**

### Q2. Mac 上要配 vLLM 吗?能用 llama.cpp/Ollama 代替?
**Mac 上不配 vLLM。** vLLM 为 CUDA 深度优化(PagedAttention 等绑定 CUDA),Mac 上装不上或只有残缺 CPU 后端 → **留到 4090**。
- **本周作业目标是"看懂内部"**(Chat Template、config 字段、tokenizer 行为、参数统计)→ 必须用 **transformers**,因为 llama.cpp/Ollama 把这些都封装隐藏了。
- **llama.cpp / Ollama** 适合"想爽快聊天/感受能力",Mac 原生 Metal 加速,但**学不到原理**。
- 结论:**这周主力 transformers 学原理;想体验 7B 顺手用 Ollama;vLLM 跳过,4090 再上。**

### Q3. M4 Pro / 16GB 能跑 7B 吗?
分方式:

| 方式 | 7B 需求 | 16GB 能否跑 |
|---|---|---|
| transformers + MPS,**fp16 全精度** | 权重 ~14GB + 系统占用 | ❌ 太挤,极易 OOM/狂 swap |
| **Ollama / llama.cpp,4-bit 量化(GGUF)** | ~4.5–5GB | ✅ 流畅,Metal 加速 |

- 16GB 是**统一内存**,macOS 还要占一部分,MPS 可用上限到不了 14GB → **7B fp16 别用 transformers 跑**。
- 想玩 7B:`ollama run qwen2.5:7b`(4-bit,~5GB),体验很好。
- **作业的 transformers 流程用 1.5B**(fp16 ~3GB,余量足,还快),架构逻辑与 7B 一致。
- 结论:**transformers 学习用 1.5B;想玩 7B 用 Ollama 4-bit。**

### Q4. Mac 上做 QLoRA 方便吗?(历史经验不如 Windows)
**不方便,几乎做不了——根因是 CUDA。**
- QLoRA 的 4-bit 量化依赖 **bitsandbytes**,而它 **4-bit 是 CUDA 专属**。之前 Windows 顺是因为有 NVIDIA 卡;Mac 无 CUDA → **QLoRA 在 Mac 上跑不起来**。
- LLaMA-Factory 也是 CUDA 导向 → **Day 5 训练必须留到 4090**(这是硬件限制,不是没配好)。
- 想在 Mac 提前摸微调手感:用 Apple 官方 **MLX**(`mlx-lm` / `mlx_lm.lora`),原生支持 Apple Silicon 的 LoRA 微调 —— 但**偏离作业指定的 LLaMA-Factory**,只当课外加餐。
- 结论:**QLoRA/训练这周不在 Mac 做,下周 4090 用 LLaMA-Factory 完成(验收❹);想 Mac 练手用 MLX。**
