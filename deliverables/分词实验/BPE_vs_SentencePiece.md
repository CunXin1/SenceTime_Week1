# BPE vs SentencePiece:中文分词深度对比 (Day 4.3)

> 对比 Qwen(byte-level BPE)与 Gemma(SentencePiece)在中文上的机制差异与实测表现。
> 数据由本地分词器真实跑出。

---

## 一、两种分词机制怎么工作

### Qwen 用的:byte-level BPE(字节级 BPE)
1. 先把文本按 **UTF-8 编码成字节**(一个汉字 = 3 个字节)。
2. 在字节序列上做 **BPE 合并**:统计高频相邻字节对,反复合并成更大的 token。
3. 显示时把字节映射到可打印字符 → 中文子词显示成**乱码**(如"你好"→`ä½łå¥½`),这只是字节的可视形式,不影响还原。
- **关键**:因为底层是字节,**任何字符都能编码,永远不会出现 `<unk>`**(未知词)——生僻字自动拆成字节。

### Gemma 用的:SentencePiece
1. 直接在 **Unicode 字符**上学习一个"子词片(piece)"词表(unigram / BPE 变体)。
2. 常见中文词直接作为一个 piece,**显示可读**(如"你好"→`你好`)。
3. 空格用 `▁` 表示;词表外的字符**回退到显式字节 token** `<0xHH>`。

## 三、罕见字 / 字节回退对比

生僻字最能看出两者底层差异:

| 字符 | Qwen(byte-level BPE) | Gemma(SentencePiece) |
|---|---|---|
| 龘(dá) | 2 个:`é¾` `ĺ`(部分字节被 BPE 合并过) | 3 个:`<0xE9>` `<0xBE>` `<0x98>`(纯字节回退) |
| 𰻞(biáng) | 4 个:`ð` `°` `»` `ŀ` | 4 个:`<0xF0>` `<0xB0>` `<0xBB>` `<0x9E>` |

- **共同点**:两者都有"字节兜底",**再生僻的字也能编码,不会崩**。
- **不同点**:
  - byte-level BPE 会把常见字节序列**合并**,所以生僻字可能只 2-3 个 token;
  - SentencePiece 对词表外字符**逐字节显式回退**成 `<0xHH>`,token 更多但更透明。

---

## 四、可逆性(都无损)

```
'你好，AI'
  Qwen  decode -> '你好，AI'   ✅ 完整还原
  Gemma decode -> '你好，AI'   ✅ 完整还原
```
两者都是**无损**的:切开再拼回,原文一字不差。乱码只是中间显示,不影响 encode↔decode。

---

## 五、对比总结

| 维度 | byte-level BPE(Qwen/Llama) | SentencePiece(Gemma) |
|---|---|---|
| 底层单位 | UTF-8 字节 | Unicode 字符片(piece) |
| 中文子词显示 | 乱码(字节可视形式) | **可读中文** |
| 空格处理 | `Ġ` 前缀 | `▁` 前缀 |
| 未知字符 | 拆成字节(无 `<unk>`) | 回退 `<0xHH>` 字节 token |
| 生僻字 token 数 | 较少(字节被合并) | 较多(逐字节回退) |
| 可逆性 | 无损 | 无损 |
| 中文 token 效率 | 高(词表大时) | 高(词表 262144 最大) |

**结论**:
1. **机制本质不同**:BPE 在**字节**上合并,SentencePiece 在**字符片**上建词表——这决定了乱码 vs 可读。
2. **中文效率取决于词表大小**,不取决于算法:我们实测同句中文平均 token,Qwen(15万词表)4.2 < Gemma(26万词表)4.4 < Llama(12.8万)5.6。
3. **两者都靠字节兜底做到无损、无 `<unk>`**,只是回退的粒度和可读性不同。
4. **实务影响**:调试分词/排查问题时,SentencePiece 的可读子词更友好;但生产中两者效果都很成熟。
